🚀 なぜNuxtサーバー経由が必要？
HttpOnly Cookie制約の完全解説

🤔 素朴な疑問

「直接ブラウザからLaravel APIを呼び出せばいいのでは？
なぜわざわざNuxtサーバーを経由する必要があるの？」

🔒 HttpOnly Cookieの2つの重要な制約

❌ JavaScript制約

JavaScriptからHttpOnly Cookieにアクセス不可

// ❌ これらは全て失敗する（HttpOnly保護） console.log(document.cookie) // → auth_jwt は表示されない const jwt = getCookie('auth_jwt') // → undefined document.cookie = 'auth_jwt=stolen' // → 設定できない（セキュリティ保護）
  • XSS攻撃からの完全保護
  • 悪意あるスクリプトもアクセス不可
  • JavaScript API完全遮断

🌐 ドメイン制約

異なるドメイン間でのCookie自動送信制限

// ❌ 異なるドメインには自動送信されない $fetch('http://nginx/api/posts/123/restore', { method: 'POST' // JWTトークンが送信されない！ }) // ✅ 同一ドメインなら自動送信 $fetch('/api/posts/123/restore') // Nuxtサーバー
  • SameSite属性による制限
  • CSRF攻撃防止機能
  • ドメイン間セキュリティ境界

🚫 直接通信が失敗する理由

❌ 直接Laravel API呼び出しの失敗パターン

🌐 ブラウザ JavaScript
「投稿を復元したい！Laravel APIを呼び出そう」
// 復元API呼び出し試行 const response = await $fetch('http://nginx/api/posts/123/restore', { method: 'POST' }) // でも認証情報をどうやって送る？
🤔 認証情報取得の試行
「JWT取得してヘッダーに追加しよう」
// HttpOnly Cookieにアクセス試行 const jwt = document.cookie .split('; ') .find(row => row.startsWith('auth_jwt=')) ?.split('=')[1] console.log(jwt) // undefined（HttpOnly保護）
⚡ Laravel API
「認証情報がないのでリクエスト拒否」
// Laravel側での認証チェック $jwt = $request->cookie('auth_jwt'); if (empty($jwt)) { return response()->json([ 'success' => false, 'error' => '認証が必要です' ], 401); } // 結果：401 Unauthorized

✅ Nuxtサーバー経由が成功する理由

✅ プロキシ経由の成功パターン

🌐 ブラウザ → 🚀 Nuxtサーバー
「同一ドメインなのでHttpOnly Cookie自動送信」
// Nuxt API呼び出し（同一ドメイン） const response = await $fetch('/api/posts/123/restore', { method: 'POST' // Cookie: auth_jwt=xxx が自動送信される })
🚀 Nuxtサーバー内処理
「サーバーサイドでHttpOnly Cookie読み取り可能」
// server/api/posts/[id]/restore.post.ts export default defineEventHandler(async (event) => { // ✅ サーバーサイドならCookie読み取り可能 const authJwt = getCookie(event, 'auth_jwt') if (!authJwt) { return { success: false, error: '認証が必要です' } } // Laravel APIにプロキシ return await $fetch(`http://nginx/api/posts/${postId}/restore`, { method: 'POST', headers: { 'Cookie': `auth_jwt=${authJwt}` } }) })
🚀 Nuxtサーバー → ⚡ Laravel API
「認証情報を含めてプロキシ転送」
// Laravel側での認証成功 $jwt = $request->cookie('auth_jwt'); // ✅ JWTを取得 $verifiedToken = $firebaseAuth->verifyIdToken($jwt); // ✅ 認証成功 // 投稿復元処理 $post->restore(); return response()->json(['success' => true]);

🔄 通信パターンの比較

❌ 直接通信（失敗）

🌐 ブラウザ → ⚡ Laravel

問題点：

  • JavaScript制約：JWT取得不可
  • ドメイン制約：Cookie自動送信されない
  • 認証失敗：401 Unauthorized
  • セキュリティ境界：越えられない
// 結果：認証エラー { "success": false, "error": "認証が必要です" }

✅ プロキシ経由（成功）

🌐 ブラウザ → 🚀 Nuxt → ⚡ Laravel

成功要因：

  • 同一ドメイン：Cookie自動送信
  • サーバーサイド：Cookie読み取り可能
  • プロキシ転送：認証情報転送
  • セキュリティ維持：HttpOnly保護継続
// 結果：復元成功 { "success": true, "message": "投稿を復元しました", "post_id": 123 }
